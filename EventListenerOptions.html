This document defines an extension to DOM events which allows authors to specify their intention not to call preventDefault on events, enabling important performance optimizations.

This proposal was produced in discussion with members of the W3C Pointer Events Working Group and the WHATWG community. It has now been integrated into the WHATWG DOM specification.

The EventListenerOptions dictionary

        dictionary EventListenerOptions {
          boolean capture = false;
          boolean passive = false;
        };
capture
Same as the capture argument to addEventListener in [[!DOM4]].
mayCancel
Indicates whether the handler might call preventDefault on the event. When all handlers for an event are installed with passive=true the user agent may make optimizations that depend on preventDefault not being called for the event. In particular, it may allow scrolling to start immediately without waiting for the event to be processed by JavaScript.

An event dispatched to a handler which was installed with passive=true will have cancelable=false for the duration of the handler, and so any calls to preventDefault will be ignored.

This property is valuable for stable composition of handler behavior. It ensures that a passive=true handler which accidentally triggers code that invokes preventDefault behaves consistently regardless of what other handlers happen to be installed for the event. As a result, passive can reliably be adopted incrementally. One component being upgraded to leverage passive=true doesn't risk exposing bugs in other components not necessarily under control of the same author.

User agents are encouraged to generate a console warning or other debugging aid to help authors identify places in their code where preventDefault is being ignored.

Extensions to the EventTarget interface

EventTarget is defined in [[!DOM4]].

        partial interface EventTarget {
          void addEventListener(DOMString type, EventListener? callback, EventListenerOptions options);
          void removeEventListener(DOMString type, EventListener? callback, EventListenerOptions options);
        };
addEventListener

Append an event listener to the list as defined by [[!DOM4]] including the specified options unless a handler with the same type and options already exists. Instead of a capture variable, each event listener has an options variable which includes capture and passive members.

This means addEventListener(type, handler, capture) is identical to addEventListener(type, handler, {capture: capture}).

removeEventListener

Remove the event listener as defined by [[!DOM4]] if one exists with the specified type and options.

Examples

The following are examples that demonstrate how the APIs in this specification might be used

        var lastTouchTime;
        document.addEventListener('touchstart', function(e) {
          lastTouchTime = Date.now();
        }, {passive: true});

Accidentally passing an EventListenerOptions as the 3rd argument to addEventListener in a user agent that doesn't support it will silently coerce the value to true, resulting in a capturing handler being installed. Authors are encouraged to rely on a polyfill for EventListenerOptions to avoid confusion here. Alternately feature detection can be done manually as follows.

        var supportsPassive = false;
        document.createElement("div").addEventListener("test", function() {}, {
          get passive() {
            supportsPassive = true;
            return false;
          }
        });
        var options = useCapture;
        if (supportsPassive)
          options = {passive: true, capture: useCapture};
        document.addEventListener('touchstart', touchHandler, options);
